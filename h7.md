### Schneier (2015) – Applied Cryptography

Chapter 1 – Foundations

#### 1.1 Terminology (Historical Terms → end)
- Cryptography is about protecting information so that only the intended person can read it.
- A plaintext is the normal readable message before encryption.
- A ciphertext is the encrypted version of the message.
- Encryption means turning plaintext into ciphertext.
- Decryption means turning ciphertext back into plaintext.
- A cipher is the method or algorithm used to encrypt and decrypt messages.
- A key is a secret value that controls how the encryption works.
- Modern cryptography usually assumes that attackers know the algorithm but do not know the key.
- Cryptanalysis is the study of breaking encryption systems or trying to recover the original message without the key.
- In the past, many simple ciphers were used, but today cryptography relies much more on mathematics and computers.

  
#### 1.4 Simple XOR

- XOR is a simple operation used a lot in cryptography.
- It works on bits (0 and 1).
- The XOR rules are simple:

0 XOR 0 = 0
<br>
1 XOR 0 = 1
<br>
0 XOR 1 = 1
<br>
1 XOR 1 = 0

- A useful property of XOR is that applying the same key twice returns the original value.
- Example:
<br>
plaintext XOR key → ciphertext
<br>
ciphertext XOR key → plaintext
<br>
<br>
- Because of this property, XOR can be used both for encryption and decryption.
- If the key is completely random and used only once, the result is called a one-time pad, which is theoretically impossible to break.

#### 1.7 Large Numbers

- Modern cryptography depends heavily on very large numbers.
- Some mathematical problems are easy to do in one direction but very hard to reverse.
- For example, multiplying two large prime numbers is easy for a computer.
- However, finding the original prime numbers from the result (factoring) is very difficult.
 Many public-key encryption systems rely on this kind of problem.
- Because computers can handle very large numbers, cryptographic systems use extremely large values to stay secure.
