### 1. Watch a value change in a loop

In this task, I used LLDB to observe how a variable changes during a loop in a Rust program. First, I built the program using `cargo build` so that it included debugging information. After that, I opened the program in the debugger using `rust-lldb`.
<br>
<br>
Inside LLDB, I set a breakpoint at the `concat` function using b concat and started the program with `run`. When the program stopped at the breakpoint, I used the `next` command to step through the code line by line. After each step, I used `frame variable n` to inspect the value of the variable `n`.
<br>
<br>
By repeating these commands, I was able to observe how the value of `n` changed during the loop execution. The value decreased step by step until it reached zero, which showed how the program updated the variable inside the loop. This demonstrated how LLDB can be used to inspect program state and follow how variables change while a program is running.

### 2. Modifying a function argument and return value
n this task, I used LLDB to modify the behavior of a Rust program during runtime. First, I built the program using `cargo build` and opened it in the debugger with `rust-lldb ./target/debug/check-pin`.
<br>
<br>
Inside LLDB, I examined the source code using `l 1` and found the correct PIN value defined in the program. Then I set a breakpoint at the function `check_password` using `b check_password`. I also disabled the previous breakpoint so the debugger would stop only at the password checking function.
<br>
<br>
When the program ran and asked for input, I entered a random PIN and the debugger stopped at the `check_password`function. At this point, I inspected the program state and then modified the return value of the function by writing `1` to the register `x0` using `register write x0 1`.
<br>
<br>
Finally, I continued the execution with `continue`, and the program printed `"Access granted!”` even though the PIN entered was incorrect. This showed that modifying the return value during execution successfully changed the program’s behavior.

### 3. Modifying a function argument

In this task, I used LLDB to modify a function argument during runtime in the `check-pin` program. I set a breakpoint at `check_password` and ran the program until it stopped inside that function. 
<br>
<br>
I inspected the arguments using `frame variable, which showed the entered PIN `(lhs)` and the correct PIN `(rhs)`. I then modified the local variable using `expr lhs = 1235` to change the function argument.
<br>
<br>
I also experimented with register manipulation by reading and writing the argument register using `register read x0` and `register write x0 1235`. This showed that function arguments can be modified both at the variable level and directly in CPU registers while the program is running.

### 4. Comparison between GDB and LLDB

- In class we used GDB (rust-gdb) to debug Rust programs and modify values during runtime.
- GDB felt simpler, because many commands are shorter and easier to type.
- LLDB commands are more strict, for example using register read and register write.
- LLDB worked better with Rust on my MacBook (ARM) and handled registers correctly.
- Overall, I would use LLDB for Rust debugging on macOS, and GDB for general debugging on Linux.
